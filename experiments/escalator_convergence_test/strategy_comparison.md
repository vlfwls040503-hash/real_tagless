## 전략별 해석

### 전략 A (Funnel geometry) — 추천 ★
- funnel 설치만으로 진동 **40% 감소** + 처리량 **증가**
- `A_funnel_5m_15°`가 최적: 멀리서부터 완만한 기울기가 최선
- 각도 30°는 효과 적음 → **완만한 각도가 유리** (Liao 2023 실험 결과와 일치)
- DO NOT REVERT 충돌 없음 (기하구조만 변경)
- **구현 비용 최소**

### 전략 B (국소 속도 감소) — 구현 실패
- JuPedSim 1.3.x의 Agent 객체에서 desired_speed 런타임 수정 실패
- `agent.model.desired_speed = v * scale`이 작동하지 않음
- 대안: spawn 시점 v0를 영역 기반으로 초기 할당하거나, waypoint + 재생성 방식으로 재구현 필요
- **후속 과제**

### 전략 C (Repulsion 파라미터 재캘리브레이션) — 부분 추천
- **strength 약화 (×0.5)**: heading 59% 감소, 처리량 유지 — 긍정적
- **strength 강화 (×1.5, ×2.0)**: heading 62~112% 증가 — 역효과 (강한 반발이 진동 악화)
- **range 축소 (×0.5)**: heading **78% 감소** (AVM 수준), backward 36% 감소 — 가장 효과적 단일 파라미터 변경
- **range 확대 (×2.0)**: heading 275% 증가 — 큰 역효과
- 결론: **range_neighbor를 0.1 → 0.05로 축소**하면 진동이 AVM 수준으로 완화됨 (그러나 density_max 증가)

### 전략 D (AVM 모델 교체) — 조건부 추천
- heading 79% 감소, p90 87% 감소 — **최고 진동 억제**
- backward 50% 감소
- **하지만 처리량 32% 감소** (34 → 23) + speed_zone 44% 감소
- 원인 추정: AVM의 anticipation + following 메커니즘이 고밀도에서 과도하게 감속 유도
- JuPedSim 공식 지원이라 구현은 쉬움
- 단방향 수렴 병목에서 AVM 파라미터 튜닝이 필요 (기본값이 bidirectional 위주)

### 전략 E (dt 감소) — 비추천
- 0.05 → 0.025로 시간 분해능 2배 상승했으나, 진동 **오히려 56% 증가**
- 해석: CFSM의 등방성 repulsion은 dt가 작아져도 해소 안 됨. 오히려 고해상도로 진동이 더 포착됨
- 계산시간 2배 증가만 초래

### 전략 F (A + E 조합) — 조합 효과 없음
- F = A_funnel_3m_15° + dt=0.025
- A 단독 (162.1) 대비 F는 177.9 → E의 악영향이 A 효과를 일부 상쇄
- **A 단독이 더 우수**

## 미시적 관측 (후속 조사)

- 전략 A와 C_range_×0.5는 **메커니즘이 다름**:
  - A: 에이전트를 물리적으로 정렬 → 진동 감소

- 둘을 **조합**했을 때(F') 시너지 가능성. 미실험.
