### 1-2. V&V (Verification & Validation) 프레임워크

본 연구는 NIST TN 1822 (Ronchi et al., 2013)의 V&V 프레임워크를 준용한다.
기존 보행 시뮬레이션 연구의 약 80%가 체계적 V&V 없이 수행되고 있어 (Lovreglio et al., 2015),
본 연구에서는 아래 절차를 따라 모델의 신뢰성을 확보한다.

**플로우차트**: `vv_methodology_flowchart_v2_mnl.svg` 참조

### 1-3. 보행 모델 (CFSM V2) 파라미터

GCFM에서 CFSM V2로 전환한 이유:
- GCFM(힘 기반)의 진동/겹침 문제 → CFSM V2(속도 기반)로 구조적 해결
- V&V Phase 2에서 V5(dt 민감도), V6(겹침) FAIL → CFSM V2 전환 후 모두 PASS

| 파라미터 | 값 | 출처/근거 |
|---------|-----|----------|
| time_gap | 0.80 s | Seyfried (2009) 병목 유량 캘리브레이션 (원논문: 1.06) |
| radius | 0.15 m | 0.55m 게이트 통로 대응 |
| strength_neighbor_repulsion | 8.0 | Tordeux et al. (2016) |
| range_neighbor_repulsion | 0.1 m | Tordeux et al. (2016) |
| strength_geometry_repulsion | 5.0 | Tordeux et al. (2016) |
| range_geometry_repulsion | 0.02 m | Tordeux et al. (2016) |
| 희망속도 | N(1.34, 0.26), clip [0.8, 1.5] m/s | Weidmann (1993) |
| dt | 0.01 s | 수치 안정성 |

### 1-4. CFSM V2 기본 검증 결과 (Verification)

RiMEA 가이드라인 및 NIST TN 1822 기반으로 3개 기본 검증 테스트 + 겹침 체크를 수행하였다.

#### 검증 종합

| 테스트 | 기준 | 결과 | 판정 |
|--------|------|------|------|
| 자유보행 속도 | 오차 < 5% | 0.0% | **PASS** |
| 기본 다이어그램 | RMSE < 0.20 m/s | 0.134 m/s | **PASS** |
| 병목 유량 | Seyfried 대비 10% 이내 | 최대 6.6% | **PASS** |
| 겹침 체크 | 0건 | 0건 (0.0cm) | **PASS** |

### 1-3. 게이트 선택 모델 (Gao LRP)

- **효용함수**: V = ω^N · 대기시간 + ω^L · 보행시간(L1+L3)
- **보행자 성격 3종** (1:1:1 비율):
  - adventurous: 대기시간 가중 1.2 / 보행시간 가중 0.8 (대기 싫어, 먼 게이트도 감수)
  - conserved: 대기시간 가중 0.8 / 보행시간 가중 1.2 (가까운 게이트 선호)
  - mild: 균등 가중 1.0 / 1.0
- **3단계 의사결정**:
  1. 1차 선택 (3.0m): Influence Zone 진입, 전체 게이트 고려, Logit 확률적
  2. 2차 재선택 (1.7m): 접근거리 기반, Logit 확률적
  3. 3차 재선택 (0.25m): 카드 태핑 위치, 확정적 전환

### 1-6. 시나리오 설계

#### 기본 시나리오 (현행 — 겸용)
- 게이트 7개 중 1~2개를 태그/태그리스 **겸용** 운영
- 나머지는 태그 전용
- 겸용 게이트에 태그·태그리스 이용자 혼재 → 태그 이용자의 정지가 뒤쪽 태그리스 이용자 대기 유발

#### 개선 시나리오 (전용 분리)
- 겸용 게이트를 **태그리스 전용**으로 전환
- 태그리스 이용자는 전용 게이트로만 진입 → 연속 통과 가능
- 태그 이용자는 나머지 게이트 이용

#### 비교 변수
- **태그리스 이용자 비율(p)**: 10%, 20%, 40%, 60%, 80%
- **태그리스 전용 게이트 수(k)**: 1대, 2대, 3대
- **시간대**: 피크 vs 비피크
- 총 시나리오: 2(기본/개선) × 5(비율) × 3(게이트 수) × 2(시간대) = **60개**
  - 각 시나리오 반복 시뮬레이션 5~10회 (확률적 모델 → 통계적 유의성 확보)
