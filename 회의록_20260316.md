# 회의록 2026-03-16

## 논의 사항

### 1. 연구 주제 방향 확정

**핵심 아이디어:**
- 현재 태그리스 게이트는 태그/태그리스 **겸용** (역당 1~2대)
- 태그리스 이용자 증가 시, 겸용 게이트에서 태그 이용자로 인해 태그리스 이용자 대기행렬 발생
- 기존 겸용 게이트를 **태그리스 전용**으로 전환하면 통행비용 절감 가능
- 서교공이 시범사업을 할 정도로 통행비용 절감 효과는 검증됨

### 2. 시나리오 설계

**기본 시나리오 (현행):**
- 대부분 태그 전용 게이트 + 겸용 게이트 1~2대
- 겸용 게이트에 태그/태그리스 이용자 혼재

**개선 시나리오:**
- 기존 겸용 게이트 1~2대를 태그리스 전용으로 전환
- 나머지 게이트는 태그 전용 유지
- 추가 인프라 변경 없이 게이트 설정만 변경하는 **현실적 방안**

### 3. 변수 설계

- **변수 1**: 태그리스 이용자 비율(p) — 10%, 20%, 40%, 60%, 80%
- **변수 2**: 태그리스 게이트 수(K) — 현행 겸용 수(1~2대)를 전용으로 전환
- **변수 3 (게이트 선택)**: 직접 설정하지 않고, Viswalk의 경로 선택 모델이 자동 산출

### 4. 게이트 선택 의사결정 모델

- 보행자의 게이트 선택은 **이산선택모형(Discrete Choice Model)** 으로 모사
- Viswalk 내장 **Partial Route + Route Choice** 기능 활용
- 효용함수: `U_gate = β₁ × 거리 + β₂ × 대기열 길이 + β₃ × 서비스 시간`
- 겸용 게이트의 태그리스 비율을 가정하지 않고 시뮬레이션 결과로 산출

### 5. 도구

- **PTV Vissim / Viswalk** 학생 버전 설치 완료
- COM API (Python 연동)는 학생 버전에서 지원 안 될 가능성 있음 → 추후 확인 필요
- 안 되면 Vissim GUI에서 직접 작업

### 6. 대상역: 성수역 (2호선)

- 환승 없음, 역사 구조 단순
- 승객/게이트 비율 상위 3위 (2,707명/게이트)
- 게이트: 턴스타일 31대 + 우대자용 8대 = 39대
