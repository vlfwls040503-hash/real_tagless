## 2-2. 게이트 처리율 vs 에스컬 부하 (핵심)

### 이론 처리율 기준 (상한 capacity)
- Pearson r = +0.109 (p=0.2820)
- R² = 0.012, 계수 = +0.957
- 해석: 이론 capacity 변화만으론 에스컬 부하 거의 변화 없음.

### 실측 처리율 기준 (`passed / SIM_TIME`)
- Pearson r = **+0.476** (p=5.729e-07)
- R² = **0.226**, 계수 = **+8.914** (s per ped/s)
- **해석: 실제 게이트가 빨리 처리할수록 에스컬 대기 길어짐 → 병목 전이 관측**

## 2-3. 최적 배합에서의 post_gate 비율

**패턴**: 각 p의 최적 config에서 post_gate 비율이 가장 큼 (55~62%). 비최적은 22~33%. 즉 **최적 배합일수록 에스컬 구간이 총 시간의 절반 이상 차지** → 게이트를 최적화하면 에스컬이 상대 병목으로 떠오름.
